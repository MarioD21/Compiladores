¿Qué es la Optimización de Código?

La optimización de código es la fase del compilador que busca mejorar la eficiencia del programa sin modificar su resultado.

El objetivo es reducir el tiempo de ejecución, el uso de memoria y el número de instrucciones necesarias.

Objetivos

  • Reducir el tiempo de ejecución.
  • Disminuir el uso de memoria.
  • Eliminar instrucciones innecesarias.
  • Mejorar el rendimiento del programa.
  • Generar código más eficiente.

Entrada y Salida

Entrada

Código Intermedio

Salida

Código Intermedio Optimizado

Tipos de Optimización

  • Optimización Local.
  • Optimización Global.
  • Optimización Dependiente de Máquina.
  • Optimización Independiente de Máquina.

Eliminación de Código Muerto

Consiste en eliminar instrucciones que nunca serán utilizadas.

Antes

x = 10;
y = 20;
z = x + y;

Después

z = x + y;

Propagación de Constantes

Antes

a = 5;
b = a + 3;

Después

b = 5 + 3;

Plegado de Constantes

El compilador resuelve expresiones constantes durante la compilación.

Antes

x = 4 * 5;

Después

x = 20;

Eliminación de Expresiones Comunes

Antes

x = a + b;
y = a + b;

Después

T1 = a + b;
x = T1;
y = T1;

Reducción de Fuerza

Sustituye operaciones costosas por otras más rápidas.

Antes

x = y * 2;

Después

x = y + y;

Desenrollado de Ciclos

Antes

for(i=0;i<4;i++)
{
   suma++;
}

Después

suma++;
suma++;
suma++;
suma++;

Optimización Local

Se realiza dentro de un bloque básico de instrucciones. 

T1 = 2 * 3;
x = T1;

Resultado:

x = 6;

Optimización Global

Analiza múltiples bloques del programa para encontrar mejoras. 

if(condicion)
{
   x = a+b;
}

y = a+b;

El compilador puede reutilizar el mismo cálculo.

Ejemplo Completo

Código Original

a = 10;
b = 20;

c = a + b;
d = a + b;

x = 4 * 5;

Código Optimizado

a = 10;
b = 20;

T1 = a + b;
c = T1;
d = T1;

x = 20;

Ventajas de la Optimización

  • Programas más rápidos.
  • Menor uso de memoria.
  • Menor consumo de energía.
  • Mayor eficiencia del procesador.
  • Mejor rendimiento general.
