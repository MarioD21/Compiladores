Inicio

¿Qué es la Generación de Código Intermedio?

La generación de código intermedio es la cuarta fase del compilador. Se encarga de transformar el árbol semántico en una representación intermedia independiente de la máquina.

Esta representación permite optimizar el programa antes de generar el código objeto para una arquitectura específica.

Objetivos

  • Generar una representación independiente de la máquina.
  • Facilitar la optimización del código.
  • Preparar la generación de código objeto.
  • Mejorar la portabilidad.
  • Reducir la complejidad del compilador.

Entrada y Salida

Entrada

Árbol Semántico

Salida

Código Intermedio

Código de Tres Direcciones

Cada instrucción contiene como máximo tres operandos. 

a = b + c * d

Se convierte en:

T1 = c * d
T2 = b + T1
a = T2

Variables Temporales

x = (a+b)*(c+d)
        
T1 = a + b
T2 = c + d
T3 = T1 * T2
x = T3

Los temporales almacenan resultados parciales durante el cálculo.

Árbol de Expresión

a + b * c

        +
       / \
      a   *
         / \
        b   c

El árbol respeta la prioridad de los operadores.

Representación por Cuádruplos

Operador Arg1 Arg2 Resultado
* c d T1
+ b T1 T2
= T2 - a

Representación por Triplos

Posición Operación Arg1 Arg2
0 * c d
1 + b (0)
2 = a (1)

Ejemplo de Traducción

resultado = a + b * c - d
        
T1 = b * c
T2 = a + T1
T3 = T2 - d
resultado = T3

Código Intermedio para IF

if(a>b)
{
    x=1;
}
        
IF a > b GOTO L1
GOTO L2

L1:
x = 1

L2:

Código Intermedio para WHILE

while(x < 10)
{
    x = x + 1;
}
        
L1:

IF x >= 10 GOTO L2

T1 = x + 1
x = T1

GOTO L1

L2:

Etiquetas (Labels)

L1:
L2:
L3:

Las etiquetas permiten controlar el flujo de ejecución.

Relación con Registros

T1 = A + B
T2 = C + D

R1 = T1
R2 = T2

Posteriormente los temporales son almacenados en registros físicos.

Ejemplo Completo

x = (a+b)*(c+d)

Código Intermedio Generado

T1 = a + b
T2 = c + d
T3 = T1 * T2
x = T3

Ventajas

  • Independencia de la máquina.
  • Facilita la optimización.
  • Mejora la portabilidad.
  • Reduce la complejidad del compilador.
  • Permite reutilizar código para distintas arquitecturas.
