¿Qué es el Análisis Semántico?

El análisis semántico es la tercera fase del compilador. Se encarga de verificar que las instrucciones tengan sentido de acuerdo con las reglas del lenguaje de programación.

Mientras que el análisis sintáctico comprueba la estructura, el análisis semántico verifica el significado del programa.

Objetivos del Análisis Semántico

  • Verificar tipos de datos.
  • Comprobar declaraciones de variables.
  • Validar el uso correcto de funciones.
  • Revisar compatibilidad entre expresiones.
  • Detectar errores semánticos.
  • Preparar la generación de código intermedio.

Entrada y Salida

Entrada

Árbol sintáctico generado por el análisis sintáctico. 

Declaración
│
├── int
├── edad
└── 20

Salida

Árbol sintáctico anotado con información semántica.

Verificación de Tipos

El compilador verifica que los tipos sean compatibles.

Correcto

int edad = 20;

Incorrecto

int edad = "veinte";

Error: No se puede asignar una cadena a una variable entera.

Uso de Variables

Correcto

int x = 10;
int y = x + 5;

Incorrecto

int y = x + 5;

Error: Variable 'x' no declarada.

Comprobación de Funciones

Función Correcta

int suma(int a, int b)
{
    return a + b;
}

Llamada Correcta

suma(5,10);

Error Semántico

suma("Hola",10);

Los parámetros enviados no coinciden con los tipos esperados.

Tabla de Símbolos

El análisis semántico utiliza constantemente la tabla de símbolos.

Identificador Tipo Ámbito
edad int Global
nombre string Local
promedio float Global

Errores Semánticos

Variable No Declarada

x = 10;

Error: La variable x no existe.

Tipos Incompatibles

int edad = "Mario";

Error: Se esperaba un entero.

División Entre Cero

int x = 10 / 0;

Error semántico detectado.

Parámetros Incorrectos

suma(5);

Error: Número incorrecto de argumentos.

Conversión de Tipos

Algunas conversiones pueden realizarse automáticamente. 

int numero = 10;
float resultado = numero;

Conversión implícita de int a float.

Árbol Semántico

x = a + b

        =
       / \
      x   +
         / \
        a   b

Durante el análisis semántico se agregan los tipos de datos a cada nodo del árbol.

Ejemplo Completo

int edad = 20;
float promedio = 9.5;
edad = promedio;

El compilador analiza si la asignación entre float e int es válida o requiere conversión.

Ventajas del Análisis Semántico

  • Detecta errores lógicos antes de ejecutar el programa.
  • Verifica compatibilidad de tipos.
  • Comprueba declaraciones y usos de variables.
  • Garantiza coherencia en el programa.
  • Reduce errores durante la ejecución.
