¿Qué es el Análisis Léxico?

El análisis léxico es la primera fase de un compilador. Su función consiste en leer el código fuente carácter por carácter y transformarlo en una secuencia de tokens que posteriormente serán utilizados por el análisis sintáctico.

También es conocido como scanner o analizador léxico.

Objetivos del Análisis Léxico

  • Leer el código fuente.
  • Reconocer tokens.
  • Eliminar espacios en blanco innecesarios.
  • Ignorar comentarios.
  • Detectar errores léxicos.
  • Enviar tokens al analizador sintáctico.

¿Qué es un Token?

Un token es una categoría léxica reconocida por el compilador. 

float promedio = 95.5;
Lexema Token
float PALABRA_RESERVADA
promedio IDENTIFICADOR
= OPERADOR
95.5 CONSTANTE
; DELIMITADOR

¿Qué es un Lexema?

Un lexema es la secuencia de caracteres encontrada en el código fuente que coincide con un patrón reconocido por el compilador. 

int edad = 20;
Token Lexema
IDENTIFICADOR edad
NUMERO 20

Expresiones Regulares

Los analizadores léxicos utilizan expresiones regulares para reconocer patrones.

Identificador:
Letra (Letra | Digito)*

Número Entero:
[0-9]+

Número Real:
[0-9]+\.[0-9]+

Errores Léxicos

Carácter no reconocido

int @edad = 20;

Error: símbolo '@' no válido.

Identificador inválido

int 2edad;

Error: un identificador no puede iniciar con números.

Cadena sin cerrar

string nombre = "Mario;

Error: cadena de texto no terminada.

Ventajas del Análisis Léxico

  • Detecta errores desde las primeras etapas.
  • Reduce la complejidad del compilador.
  • Mejora la velocidad de compilación.
  • Facilita el análisis sintáctico.
  • Organiza el código en tokens.
